# Contributing

Thank you for investing your time in contributing to the project!

## General Rules

Before contributing to the project, please, review the existing code and docs; try to follow this format as much as possible.

Tests are not required, however, try to cover your code by tests as much as possible.

Documenation is important; please, updates docs if you add a new functionality or change an existing one.

Comment your code even if it looks obvious.

## Issues

If you spot a problem, search if an issue exists. Open a new issue, only if a related issue doesn't exist.

If you find a typo, a broken link, or you want to fix a sentence, click **Make a contribution** button at the bottom of any docs page. This takes you to the `.md` file where you can make your changes and create a pull request for a review.

If you find an existing issue in code that interests you, you're welcome to create a pull request with a fix.

## Pull Requests

Before submiting a pull request, please, run `npm run lint` to ensure your code is well formatted.

Please, be specific commenting your pull request, so reviewers could understand your changes as well as the purpose of your pull request.

Dont' forget to link a pull request to an issue if you're solving one.

If you run into any merge issues, checkout this [git tutorial](https://lab.github.com/githubtraining/managing-merge-conflicts) to help you resolve merge conflicts and other issues.
